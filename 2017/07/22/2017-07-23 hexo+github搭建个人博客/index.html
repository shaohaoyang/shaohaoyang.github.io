Hexo+github 搭建博客

发表于 2017-07-22 | 分类于 github | 阅读次数 次

现在可以发博客的地方非常多，自己写的一些东西往往会散布在各个地方，这样做一方面不容易集中管理，另一方面可能哪家公司关停了自己的相关业务，自己发的东西也会遗失。所以在github上对自己的文章进行管理就显得相当必要。下面就简要介绍使用hexo和github搭建自己个人博客的流程。

一、博客搭建流程

github相关

  • 申请github账号，使用免费账户即可
    • 申请账户时建议username使用自己的全名或者自己喜欢的英文写法. 因为最终博客的地址就是 www.username.github.io
    • 如果之前已经有github账号，可以在setting中的Account中Change username
  • 新建repository
    • repository name 必须填写成自己的 username.github.io, 因为github通过这种格式的命名来确定用户是要建立自己的页面
    • repository的类型可以选择public, 可以勾选initialize
    • 可以搜索github pages来了解更多，有一种方法就是在这步之后进入到此项目的setting里，找到github pages，通过Choose a theme来设置首页。这种方法是通过Jekyll+github的方式来进行的，也相当方便，感兴趣的可以搜一下

  • 本地安装git

    • 直接使用git bash就行
    • 配置github账户,在gitBash里输入如下语句，其中username填写注册github时的账户，邮箱填写注册时的邮箱

  • 配置SSH

    1. 打开gitBash,输入 ssh-keygen -t rsa -C “your e-mail” 生成SSH，这里的your e-mail填写注册github时的邮箱，然后一路按回车即可
    2. 打开 ~/.ssh/id_rsa.pub 文件，复制其中的内容。打开github, 点击头像进入setting，在左侧边栏找到SSH相关选项卡，点击Add SSH key, 填写title，这个title自己可以随便起一个名字，然后将复制的内容粘贴到key框中，最后点击add key即可
    3. 输入 ssh -T git@github.com，如果终断里显示如下语句则表示配置成功

hexo相关配置

  • 安装hexo

    • 下载Node.js并安装。需要配置好环境变量，安装完成后可以在终端里输入node --version查看是否有效
    • 打开gitBash,输入 npm install -g hexo-cli 完成hexo的安装

    • 在本地新建一个供自己存放项目的文件夹，例如C:\hexo，在gitBash中切换到此文件夹，输入下边两句命令即可安装好本地的hexo

    • 接着输入 hexo server 就会启动本地的博客网站，默认地址是 http://localhost:4000/

      • 通过4000端口启动的是本地的博客，这个是用来查看自己本地当前的博客是什么样的，供调试使用。如果本地查看完后觉得可以，通过后续的编译上传，可以通过github的链接直接查看，网站上的查看是本地改好上传的。
      • 在浏览器中输入 http://localhost:4000/ 如果打不开，则有很大的原因是4000端口被占用，可以输入hexo s -p 5000，来设置5000端口，或者自定义一个自由端口即可
      • 每次修改完毕之后，可以执行 hexo s --debug, 或者指定端口 hexo s -p 端口号 --debug来进行本地调试
  • 切换主题
    hexo默认带有一个主题，在其themes文件夹下，如果对当前主题不满意，可以切换自己喜欢的主题。通过搜索“hexo主题”这个关键词可以搜到新的主题。本博客现在使用的主题是NEXT主题，下面简要介绍下下载主题后的配置。

二、搭建过程中可能遇到的问题

  1. 添加tags, categories, about后跳转失败
    问题原因是博客，tags,about,categories需要单独维护，也可以对其内容进行填充，所以需要单独建立页面，使用命令hexo new page tags就可以新建tags页面，会在source目录下生成一个tags目录，about和categories也类似，about中也可以填充自己要进行介绍的说明
